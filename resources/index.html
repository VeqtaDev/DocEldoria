# Ressources Eldoria — descriptions

Brèves descriptions des ressources du serveur FiveM Eldoria (Magic Verse), déduites de l’analyse du code.
Les ressources sont regroupées par dossiers [admin] et [body].

# [admin]

# hgs_commands

  • Rôle : Commandes admin (superadmin) pour le serveur.
  • Contenu : main.lua enregistre toutes les commandes (tp, car, giveitem, giveweapon, givecard, ban, setjob, skin, revive, etc.) ; lib.lua expose la fonction command() et vérifie le groupe (superadmin) ; client.lua gère la téléportation, le spawn de véhicules, la suppression de véhicule et le changement de modèle.
  • Dépendances : ESX, hgs_datacenter (resetban), hgs_copything (coords), hgs_log (logs des commandes).
  • Chemin : resources/[admin]/hgs_commands/src/ (sources) ; build via build.bat → build/sexy.lua.

# hgs_canuseeme

  • Rôle : Vérification / gestion des autorisations d’utilisation (probablement anti-abus ou whitelist).
  • Contenu : Client dans src/client.lua, build via build.bat.
  • Chemin : resources/[admin]/hgs_canuseeme/.

# [body] — Scripts gameplay

# hgs_card

  • Rôle : Système de cartes (gacha / collection) : ajout, suppression, activation, surnom, vente.
  • Contenu : list_card/card_1.lua à card_12.lua définissent les cartes (name, skill, level_exp, img). Client : card.add, card.update. Serveur : card.addNewCard, card.ClearAllCard, card.changeName, card.ActivateCard, callbacks getAllCard, SellCard.
  • Dépendances : hgs_lib, es_extended callback, hgs_fixnui.
  • Lien doc : Cartes.

# hgs_commands (côté client)

  • Rôle : Implémentation client des commandes admin : tp, tpm, spawn véhicule, delete véhicule, changement de modèle ped.
  • Voir : Commandes.

# hgs_equipment

  • Rôle : Gestion des équipements (armes, armures, etc.) : liste dans list/*.lua, équipement client, calcul de statuts côté serveur (status_cal.lua).
  • Contenu : Fichiers list pour armes (épées, arcs, lances, harpes, etc.), client/equip.lua, playe_state.lua.
  • Dépendances : es_extended callback, hgs_lib, oxmysql.
  • Lien doc : Armes.

# hgs_skill

  • Rôle : Système de compétences / sorts (skills) : config par skill dans skill_config/*.lua (ex. chester_dash, slam_dash, storm, meteorite, fire_wall, etc.), handlers, skeleton_class.
  • Contenu : Client uniquement (init, config, function, skeleton_class, skill_config, skill_handler).
  • Dépendances : hgs_lib, hgs_fixnui.

# hgs_quest

  • Rôle : Système de quêtes et missions : progression, statuts, objectifs, expiration.
  • Contenu : Client et serveur avec dossiers quest/**, config.lua, expire.lua ; events quest.addquest, quest.removequest, quest.queststatus, quest.setmisstionvalue, quest.clearallquest.
  • Dépendances : es_extended callback, hgs_lib, hgs_vPrompt, oxmysql.
  • Lien doc : Commandes — Quêtes.

# hgs_shop

  • Rôle : Boutique in-game : catégories (Sword, Sicsickle, Staff, Fan, Spear…), items avec prix et type (equip, item, card), thèmes (green, blue, gray).
  • Contenu : config.lua (h.Items avec Category / Item), client/serveur dans main/, NUI (dist).
  • Dépendances : es_extended callback, hgs_lib, oxmysql.
  • Lien doc : Items, Armes.

# hgs_donate

  • Rôle : Boutique donateur / premium : packages, skin, gashapon, limited, store, topup, VIP ; NUI (dist) et images (img).
  • Contenu : Shop scripts dans shop/*.lua, vip.lua, client/serveur, events pour donner items/cartes/skins.
  • Dépendances : es_extended callback, hgs_lib, oxmysql.
  • Lien doc : Items, Cartes.

# hgs_crafting

  • Rôle : Craft d’objets : interface NUI (dist), config, client/server pour combiner composants et obtenir items.
  • Dépendances : es_extended callback, hgs_vPrompt, hgs_lib.
  • Lien doc : Items (types item_component).

# hgs_inventory

  • Rôle : Inventaire joueur : catégories (inventory), vente, coffre (safebox), gestion des items ESX/db_item.
  • Contenu : Category inventory (client/server), config, dist (NUI).
  • Dépendances : ESX Items (db_item dans es_extended/item.lua).
  • Lien doc : Items.

# hgs_safebox

  • Rôle : Coffres de stockage : dépôt / retrait d’items, synchronisation avec l’inventaire.
  • Contenu : Client/server function, utilisation de ESX Items.
  • Lien doc : Items.

# hgs_collect

  • Rôle : Collecte d’objets dans le monde (ramassage, récoltes) : config, client/server, images.
  • Dépendances : ESX, hgs_lib.
  • Lien doc : Items.

# hgs_skin

  • Rôle : Gestion des skins (apparence) : menu skin, ajout/retrait via events on.addskin.menu, on.removeskin.menu, config, NUI.
  • Contenu : Client (function, client, nui), server (function), config.
  • Lien doc : Commandes — Skins.

# hgs_clothingmenu

  • Rôle : Menu d’habillement (vêtements, accessoires) : events cloths:command pour add/remove cloth par type et ID.
  • Lien doc : Commandes — addcloth / removecloth.

# hgs_deathscreen

  • Rôle : Écran de mort / résurrection : affichage NUI, événements on.revive, on.heal.
  • Lien doc : Commandes — revive / heal.

# hgs_damage

  • Rôle : Gestion des dégâts (joueur, armes, skills) : client/server, export pour calculs.
  • Dépendances : Armes et skills (hgs_equipment, hgs_skill).

# hgs_monster

  • Rôle : PNJ / monstres : spawn, IA, récompenses, possible lien avec quêtes et cartes (givecard).
  • Contenu : skeleton_class (monster.lua), client/server.
  • Dépendances : hgs_lib.

# hgs_ride

  • Rôle : Monture / véhicule spécial (équitation ou véhicule personnalisé) : client (client.lua, function.lua), utilisation d’items/armes.

# hgs_transform

# hgs_wing

# hgs_capsule_gashapon

# hgs_stone

# hgs_exp

# hgs_expire
